1. TinyMCE — classic iframe mode (the failing case)

Default TinyMCE setup. The editable area lives inside an <iframe> (its body is contentEditable). Content scripts injected into the top page do not automatically run inside this iframe — this is the case issue #2 is about.

2. TinyMCE — inline mode (no iframe)

Inline mode replaces a div in the parent document instead of using an iframe. If TextFast's keyboard handler runs on the top page, this one should work even without iframe support.
